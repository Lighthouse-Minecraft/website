## Overview

Brigged users can submit appeals to request their access be restored. Appeals create support tickets in the **Quartermaster** department. When you're ready to restore a user's access, you release them from the Brig.

## How Appeals Work

When a brigged user submits an appeal:

1. They write an appeal message (minimum 20 characters) from their Dashboard
2. The system creates a **Quartermaster ticket** with their message
3. Quartermaster staff receive a notification about the new ticket
4. A **7-day cooldown** starts before they can appeal again
5. The appeal appears in the ticket queue like any other ticket

Appeals are a way for users to explain their side and request another chance. Treat them like any support ticket -- read the appeal, discuss with the team if needed, and decide whether to release or keep the restriction.

## Releasing a User from the Brig

### Who Can Do This

- Admins
- Officers (any department)
- Crew Members in the **Quartermaster** department

### How to Release

1. Navigate to the user's **profile page**
2. Click the **three-dot menu** on their profile card
3. Select **Release from Brig**
4. Enter a **reason** for releasing them (minimum 5 characters)
5. Click **Confirm**

## What Happens on Release

When you release a user, the system automatically:

- Clears their brig status and reason
- **Restores their Minecraft accounts** -- previously active accounts are re-whitelisted on the server
- **Restores their Discord roles** -- membership level and department roles are re-synced
- Records the release in the **activity log**
- Sends the user a **notification** that they've been released

## Parental Hold After Release

If a released user is a minor with a linked parent account, the system checks whether the parent has restricted their access. If the parent previously disabled the child's account, the user may be immediately re-brigged with a **Parental Disabled** brig type. This is automatic -- you don't need to do anything, but don't be surprised if a released minor shows back up in the Brig.

## Important Notes

- Always coordinate with your team before releasing someone, especially for serious infractions
- The release reason is logged and can be reviewed later -- be clear about why the decision was made
- Promote/demote options are hidden while a user is in the Brig. Once released, you can manage their membership level normally
- The system handles brig types other than disciplinary (parental, age lock) automatically. You'll typically only deal with the **Discipline** brig type as staff
