## Overview

The **Brig Warden Widget** appears on your staff Dashboard when you have the Brig Warden role. It gives you a quick view of who's in the Brig, which users have timers expiring soon, and how many open appeals need attention -- without having to search through individual profiles.

## Who Sees This

Only staff with the **Brig Warden** role see this widget. It's displayed in the Staff Dashboard section of your Dashboard.

## What the Widget Shows

The widget has three key indicators at the top:

- **Open Appeals** -- the number of open Brig Appeal threads. This badge turns red when there are pending appeals waiting for review. Click it to go to the discussions list.
- **Total in Brig** -- the total count of all users currently brigged (shown in amber).
- **View All** -- opens a searchable table of every brigged user.

Below those badges, the widget lists users with a **brig timer expiring within 7 days**, sorted by expiration date. Each row shows the user's name, their discipline risk score (color-coded by severity), and the time remaining on their timer.

## Managing a User from the Widget

### From the Approaching Release List

1. Find the user in the approaching release list
2. Click **Manage** next to their name
3. The **Brig Status Manager** opens in a modal -- use it to update their reason, timer, or release them

### From the View All Modal

1. Click the **Total in Brig** badge or **View All** button
2. Search by name using the search field
3. Sort the table by clicking any column header (Name, Type, Date Placed, Expires At)
4. Click **Manage** on any row to open the Brig Status Manager for that user

The View All modal shows all brigged users -- including permanent ones (marked with a **Permanent** badge) and parental/age-lock types (shown in blue). The list is useful for getting a full picture of who's currently restricted.

## Important Notes

- Parental and age-lock brig types (shown in blue) are managed automatically by the system -- you typically don't need to take action on them
- Users with no expiry timer don't appear in the approaching release list, but they still show up in the View All modal
- The approaching release list excludes permanently confined users, since those have no timer to expire
