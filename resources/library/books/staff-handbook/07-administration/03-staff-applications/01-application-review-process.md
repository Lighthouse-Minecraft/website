## Overview

When a community member applies for a staff position, their application moves through a structured pipeline. Command Officers and Admins are responsible for guiding each application through the process and making a final decision.

## Who Can Review Applications

- **Admins** -- full access to all review actions
- **Command Department Officers** -- full access to all review actions
- **Crew Members** (any department) -- can view the review list and application details, but cannot take any actions

## The Application Pipeline

Every application moves through these statuses:

| Status | What It Means |
|---|---|
| **Submitted** | The applicant just sent in their application. It hasn't been looked at yet. |
| **Under Review** | A reviewer has opened the application. This happens automatically the first time a Command Officer or Admin views it. |
| **Interview** | The applicant has been selected for an interview. A discussion thread is created automatically for this. |
| **Background Check** | The applicant is going through a background check. |
| **Approved** | The application was accepted. The applicant is automatically assigned to the position. |
| **Denied** | The application was not accepted. The applicant is notified. |
| **Withdrawn** | The applicant pulled their own application before a decision was made. |

Once an application reaches Approved, Denied, or Withdrawn, it's final -- no further changes can be made.

## What Happens at Each Stage

### Submission

When someone submits an application:

- The system captures a snapshot of their profile at that moment (age, membership level, how long they've been a member, report and commendation counts)
- A **staff-only review discussion** is automatically created and the relevant staff members are added as participants
- You and other Command Officers/Admins receive a notification about the new application
- The applicant receives a confirmation notification

### Under Review

The first time a Command Officer or Admin opens the application detail page, the status automatically moves from Submitted to Under Review. The applicant is notified of this change.

### Interview

When you schedule an interview:

- An **interview discussion thread** is created automatically
- The applicant is added to the discussion so they can participate
- Officers and department crew members are also added
- The applicant is notified that they've been selected for an interview

### Background Check

When you move an application to Background Check, you'll set the initial background check status (Pending, Passed, Failed, or Waived). You can update this status independently at any time without changing the application status.

### Approval

When you approve an application:

- The applicant is **automatically assigned** to the staff position
- Both the staff review and interview discussions are closed and locked
- The applicant receives an approval notification

### Denial

When you deny an application:

- Both the staff review and interview discussions are closed and locked
- The applicant receives a notification (worded sensitively -- the subject says "Application Update" rather than "Denied")

## Notifications

The applicant receives an email notification at every status change. Reviewers only receive a notification when a new application is submitted -- not for every status change.

## Important Notes

- Applicants can only have **one active application** at a time across all positions
- If an applicant was previously denied, they can reapply later
- Applicants can withdraw their own application at any point before a final decision
- The applicant snapshot (age, membership level, etc.) is frozen at submission time -- it won't change if their profile updates later
- All review actions are logged in the activity log
